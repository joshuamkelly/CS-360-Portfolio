# CS-360-Portfolio

The application I have created for the course is an Android-based inventory tracker. The main purpose of the app is to provide users with an opportunity to manage their inventory items in an easy way from their mobile devices. Users will be able to create an account, log in, add inventory items, view their saved inventory records, update quantities of certain items, delete items and configure SMS notifications regarding low stock level alerts. This application is meant for those people who need an easy tool for managing supplies, without the need to write the inventory down with a pencil or update a spreadsheet separately.

To satisfy the user's needs several screens should be added to the app. The login screen will provide users with an opportunity to create an account or sign in. The inventory dashboard will show the user the current state of his saved inventory records. The add-item screen will enable the user to add item name, quantity, and status. The SMS notification screen will let users configure the notifications according to their preferences. In order to make the UI clear and easy to use, I have assigned titles, descriptions and buttons according to the user's next steps. This decision proved to be right, because the design did not contain unnecessary elements and provided users with an opportunity to do the things they needed.

My approach to coding process was to break up the application into smaller parts and implement features individually. I first worked with the existing UI design, connecting screens to the app's functionality. I have created a database helper class, responsible for all SQLite-related operations and connected the login screen, inventory screen and add-item screen with this database. This solution helped to organize the code properly as all database operations were implemented in one place. I have tested the main feature after its implementation, rather than testing everything at once.

For the purpose of testing the application I have used the Android emulator in Android Studio. I have tested the login function, by creating a new account and logging in with saved credentials. For the inventory database I have tested the adding of a new item, its appearance on the dashboard, updating the quantity with plus and minus buttons and deleting the item from the list. At last I have tested the SMS notification screen by picking allow option and no-thanks option. Testing is very important because even if an app looks good on the design stage, there could be errors during the user interaction. This process showed the problems with layout, database connections and problems with buttons' visibility and usability.

The problem I had to solve was how to implement inventory dashboard in an easy and convenient way, without complicating the app's structure. Instead of using a complex RecyclerView setup I have chosen a dynamic layout approach in which inventory rows are created and displayed according to SQLite database. This solution allowed me to display database records without complicating the app's structure too much. Another problem I had to solve was the placement of action buttons within the inventory grid. The update and delete buttons were initially crowded, so I decided to make them smaller in order to fit the row properly. Finally, I had to deal with Android Studio issues in which generated resource references were not updated until I clean and rebuild the project.

In my opinion the most successful part of the application is the database functionality. The app uses SQLite in order to store user accounts and inventory records, thus providing database persistence. I managed to connect the database to multiple screens and implement the main functions like adding items, viewing saved items, updating quantities, automatic changing of item status and deleting items. This part of the project shows my skills in mobile app development the best.
